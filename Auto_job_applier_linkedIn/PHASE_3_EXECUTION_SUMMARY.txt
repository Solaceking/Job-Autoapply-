═══════════════════════════════════════════════════════════════════════════════
  ARCHITECTURE HIGHLIGHTS
═══════════════════════════════════════════════════════════════════════════════

Signal/Slot Flow:
  LinkedInSession
    └─ JobApplicationManager.apply_to_job()
       └─ emit progress_callback(applied, failed, skipped, job_title)
          └─ AutomationWorker.progress_signal.emit()
             └─ MainWindow._on_worker_progress()
                └─ Update labels & progress bars

Config Loading Flow:
  fill_application_form()
    ├─ load_personals_settings() → dict
    ├─ load_resume_settings() → dict
    ├─ merge with provided form_data
    └─ pass merged_data to FormHandler & QuestionHandler

Field Matching Flow:
  FormHandler.fill_form()
    ├─ detect_fields() → collect label candidates
    ├─ for each field:
    │  ├─ try exact key match
    │  ├─ try normalized match
    │  ├─ try token-overlap match (60% threshold)
    │  └─ fill_[text|select|checkbox|file]()
    └─ return result dict with per-field status

Answer Matching Flow:
  QuestionHandler.match_answer()
    ├─ normalize question text
    ├─ find exact normalized key match
    ├─ score all keys using token overlap
    ├─ return (answer, score) for best match
    └─ caller decides if score >= min_score
